# 🧠 基于LLM的个人知识库

欢迎使用你的个人知识库系统！这个系统使用大型语言模型（LLM）作为wiki维护者，将原始资料编译成结构化的、相互链接的知识网络。

## 🚀 快速开始

### 1. 添加源文档
将你的源文档（文章、论文、书籍等）放入相应的目录：
- `raw/articles/` - 文章和博客帖子
- `raw/papers/` - 学术论文  
- `raw/books/` - 书籍章节
- `raw/transcripts/` - 会议记录、播客文字稿
- `raw/images/` - 图片资源
- `raw/assets/` - 其他资产

### 2. 处理源文档
告诉LLM处理新文档：
```
请处理 raw/articles/你的文章.md
```

LLM将会：
1. 阅读并理解内容
2. 创建摘要页面 (`wiki/sources/`)
3. 更新相关实体和概念页面
4. 更新索引和日志

### 3. 探索知识库
- **浏览wiki**：查看 `wiki/` 目录中的页面
- **使用索引**：查看 `wiki/index.md` 了解所有页面
- **查看关系**：使用Obsidian的图形视图查看页面连接
- **搜索内容**：运行 `python scripts/search.py "关键词"`

### 4. 提出问题
向LLM提问关于知识库的问题：
```
"人工智能的主要伦理问题有哪些？"
```

LLM会搜索相关wiki页面并给出综合回答，引用相关页面。

## 📁 系统架构

```

```

## 🔧 核心功能

### 📥 摄取流程
- 一次处理一个源文档，保持人类参与
- 自动更新相关页面
- 维护交叉引用和一致性

### 🔍 智能查询
- 基于wiki内容而非原始文档回答
- 回答可保存为新wiki页面
- 支持多种输出格式（表格、图表等）

### 🧹 自动维护
- 定期健康检查（lint）
- 检测矛盾和过时信息
- 建议改进和缺失内容

## 📚 模板系统

使用模板确保一致性：
- `templates/source_summary.md` - 源文档摘要
- `templates/entity_page.md` - 实体页面
- `templates/concept_page.md` - 概念页面
- `templates/topic_overview.md` - 主题概述

## 🛠️ 工具脚本

- `scripts/search.py` - wiki搜索工具
- `scripts/batch_ingest.sh` - 批量处理脚本

## 📖 详细文档

查看 `CLAUDE.md` 获取完整的工作流程、约定和最佳实践。

## 💡 使用技巧

1. **逐步构建**：一次处理一个源文档，检查LLM的更新
2. **验证交叉引用**：使用图形视图确保链接正确
3. **定期维护**：每周运行一次lint检查
4. **保存有价值回答**：将重要分析保存为wiki页面

## 🎯 示例工作流

1. **添加文章**：`raw/articles/ai-ethics.md`
2. **处理文档**：告诉LLM处理该文件
3. **检查更新**：浏览新创建的wiki页面
4. **提出问题**："AI伦理中的主要争议？"
5. **保存分析**：将回答保存为比较页面
